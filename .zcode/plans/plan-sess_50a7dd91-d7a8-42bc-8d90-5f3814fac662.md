## 文言文按文体分类方案

### 一、目录结构（按文体分，4 大类）

```
chinese/
├─ index.md
├─ 散文/                      ← 议论/记叙/游记/诸子/史传/奏疏/史论 都放这
│   ├─ 1-齐桓晋文之事.md      (原 1-寡人之于国也.md，正名)
│   ├─ 2-秋水.md              (原 2-庄子-秋水节选.md，简化)
│   ├─ 3-谏逐客书.md          (保持)
│   ├─ 4-过秦论.md            (保持)
│   └─ 5-五代史伶官传序.md    (原 5-五代伶官续转.md，正名)
├─ 诗词曲/                    ← 唐诗/宋词/元曲/古体近体/词牌散曲 (预留)
├─ 小说/                      ← 传奇/话本/明清小说节选 (预留)
└─ 赋骈文/                    ← 辞赋/骈文 (预留)
```

- 散文不细分（诸子/史传/奏疏/史论用 frontmatter `tags` 区分）
- 预留 3 个空目录（`.gitkeep`），等你发对应文体文章时直接归入

### 二、具体改动

1. **新建目录**：`散文/` `诗词曲/` `小说/` `赋骈文/`（后三者带 `.gitkeep`）
2. **移动 5 篇文件到 `散文/`**，其中 3 篇同时正名：
   - `1-寡人之于国也.md` → `散文/1-齐桓晋文之事.md`
   - `2-庄子-秋水节选.md` → `散文/2-秋水.md`
   - `3-谏逐客书.md` → `散文/3-谏逐客书.md`
   - `4-过秦论.md` → `散文/4-过秦论.md`
   - `5-五代伶官续转.md` → `散文/5-五代史伶官传序.md`
3. **更新 5 篇 md 内部链接**：文档里引用首页 `[进度看板](/progress)` 等是根路径，不受影响；无需改动正文。

### 三、config.mts sidebar 改造

把写死的单组 sidebar 改为**按文体分组的统一 sidebar**（用 `/chinese/` 前缀共用一份）：

```
📚 大学语文
├─ 📖 首页
├─ 📝 散文 (5)        ← 可折叠，列 5 篇
├─ 🎵 诗词曲 (0)      ← 可折叠，预留
├─ 📕 小说 (0)        ← 可折叠，预留
└─ 🎭 赋骈文 (0)      ← 可折叠，预留
```

实现方式：把 `sidebarGroup` 函数改为读取各文体目录，统一生成多分组结构。每篇登记一次即可。

### 四、自动归类规则（写入 AGENTS.md）

在项目级 AGENTS.md 新增「资料归类规范」一节，让我收到新文章时**先判文体，再归入对应目录**：

| 文章文体 | 归入目录 |
| --- | --- |
| 散文（诸子/史传/奏疏/史论/游记/记叙议论） | `chinese/散文/` |
| 诗词曲（诗/词/曲/古体近体/词牌散曲） | `chinese/诗词曲/` |
| 小说（传奇/话本/明清小说节选） | `chinese/小说/` |
| 赋/骈文（辞赋/骈体文） | `chinese/赋骈文/` |

规则要点：
- 收到新文章**先判文体** → 归入对应子目录
- 文体目录不存在则新建
- 文件名：`序号-篇名.md`（序号 = 该文体目录内已有最大值 +1）
- 新增文章后**自动登记到对应 sidebar 分组**
- 时代/流派/作者信息放 frontmatter `tags`，不进目录名

### 五、验证

`npm run build` 构建无报错，5 篇页面可访问，sidebar 文体分组正确显示。

- 不改首页 features 卡片
- 不引入自动生成插件（保持手动，清晰可控）
